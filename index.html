Abstract

LLM agents are increasingly used to translate natural language into 3D scenes in a procedural way, but existing systems focus on static output. Dynamic, simulation-ready scenes from text alone — liquids flowing, particles emitting, rigid-body cascades, and articulated mechanical motion — remain largely unexplored despite their value as editable content and as physics-grounded training data for video generation and embodied AI.

Two challenges set the dynamic case apart from static text-to-scene work: an agent must jointly coordinate spatial layout, multiple physics solvers, animation timelines, camera, and lighting in a single coherent scene; and verifying motion correctness from rendered video is fundamentally harder than judging a single image. We present SimWorlds, a multi-agent framework that produces simulation-ready dynamic 3D scenes from text, with Blender-specific procedural knowledge, a planner–coder–reviewer workflow driving a fixed ordered sequence of construction stages, a layered scene protocol enforced by an orchestrator-side verifier, and a runtime-state inspection tool suite that catches mechanism failures the rendered image cannot reveal. We also introduce 4DBuildBench, a benchmark for assessing both visual fidelity and physical consistency of procedural dynamic 3D scenes generated from text. Experiments show that SimWorlds outperforms prior dynamic Blender generation baselines.

How SimWorlds Works

A planner–coder–reviewer loop drives a fixed, ordered sequence of construction stages. The planner stays qualitative; the coder owns every number; the reviewer re-judges from rendered evidence.

modeling→ uv→ texture→ deformation setup→ motion→ camera→ light→ render

Eight ordered stages. deformation setup and motion extend the static pipeline to cover the four dynamics axes: animated camera, prop motion, physics simulation, and character animation.

🧩

Layered scene protocol

A three-level collection tree plus per-object Empty roots make "logical object," "structural contact," and "grounding" first-class concepts on top of Blender's native graph.

✅

Orchestrator-side verifier

After every state-mutating call, a verifier runs the full ruleset — structural, geometric (BVH contact & penetration), and time-axis — and blocks progress until ok=true.

🔬

Runtime-state inspection

A tool suite reads live bpy engine state to catch mechanism failures the rendered image cannot reveal — e.g. motion that is keyframed rather than simulated.

4DBuildBench

50 scenes spanning five dynamic solver categories × three difficulty levels, plus static controls — evaluated on a two-track protocol that scores any system identically.

🗂️

50 scenes, 3 levels

L1 solo-simple · L2 intra-complex (self-collision, pinning, force fields) · L3 inter-coupling (e.g. a rigid body crushing a soft body). Hand-authored ground-truth YAML per scene.

⚙️

Engine-state audit

Runs in headless Blender, system-agnostic. 42 typed predicates yield MPR (Mechanism Pass Rate) and SPR (Structural Pass Rate) — catching mechanism-cheating an image misses.

👁️

VLM judge

A vision–language judge scores five dimensions — objects present, spatial relations, actions visible, visual quality, aesthetics — as a pass fraction over atomic items.
